1. 求|sum(A) - sum(B)|最小值，`nums[i]`范围`[0, 100]`，`N`不超过10000.  
思路：记数组的总和为`total`，最理想的情况，`|sum(A) - sum(B)| ＝ 0`，即`sum(A) = halfTotal = total / 2`，若不存在，退而考虑有没有`sum(A) = halfTotal - 1`。
确定了一个集合的所有元素，也就确定了另一个集合。不妨以集合A为讨论对象，数组中的每个元素，要么加入A，要么不加入。这就跟01背包问题很类似了。
定义状态`DP(i)`为：处理到数组第`i`个元素时，`sum(A)`所有可能的取值，那么`DP(i)`将是一个集合。
所求结果：令`sum(A)`等于，众多`DP(i)`中最接近`halfTotal`的那一个值，计算结果即可
转移方程：`DP(i) = DP(i - 1) ∪ DP(i - 1) ⊕ A[i]`，其中符号`⊕`表示，左边集合中每个元素取值都增加`A[i]`。
**重新理解`动态规划DP转移方程`，不是非得写成`dp(i) = Max(dp(i - 1), sth)`式子的，才叫动态规划或者背包。**

1. 小白鼠检测毒药瓶  
题目描述：100瓶药水中有且仅有1瓶有毒，小白鼠喝毒药后24h内死亡。要在24h内查出有毒药瓶，至少需要多少只鼠？

**Analyse**  
+ 先考虑*至多*需要多少只鼠。显然取100只鼠足矣——*一只鼠分配一瓶药*，24h后一定死了1只鼠，对应的药瓶即为有毒。这个方案无脑但有效。
+ 能不能少于100呢？想想发现99只也够了——依然是*一只鼠分配一瓶药*，最后剩余1瓶药，根据另外99瓶药的检测结果，即可推断。
+ 再想，*每只鼠完全可以喝不止一瓶药*，那么通过*构造适当的喂药方案*，应当可以用更少的鼠完成任务。
+ 怎么构造呢？——接下来*猜测并验证*，log2(100)向上取整得7，7只鼠即可满足要求。

**喂药方案** 

7只鼠一字排开，从右到左依次记为`鼠0～鼠6`。100瓶药顺序摆好，记为`药瓶0～药瓶99`，依次将每瓶药选择`几只`小鼠进行投喂。那么某瓶药到底选择`哪几只`鼠进行投喂呢？这样做——计算该药瓶编号对应的二进制编码，刚好得到7个布尔值（不足7位的，在高位补0），就依据这7个布尔值来来决定对应的7只鼠是否喂药：值为1代表喂药，值为0代表不喂。比如`药瓶20`对应二进制为`0010100`，那么就只喂给`鼠2`和`鼠4`，其余不喂，因为从右向左数，只有第2、第4个位上布尔值为1。

**结论**

24h后看7只鼠的生死情况，死对应`1`，生对应`0`，得到一个7位的二进制序列，还原为十进制，就是毒药瓶的编号了。

**验证**

以10瓶药水中1瓶有毒为例，说明问题。log2(10)向上取整得4，就首先预备4只鼠。按照上述喂药方案，相当于对`药瓶0～药瓶9`全部进行二进制编码，并让`鼠0～鼠3`分别占据一个二进制位。有毒药水一定会致死喝了它的小鼠，反之没有喝它的小鼠则一定存活，那么根据鼠的生死情况还原出的0 1序列，刚好就是毒药瓶编号的二进制表示。整个过程可以理解成一种**压缩编码**，利用二进制，完成了压缩。

鼠	鼠	鼠	鼠  
4	3	2	1  
瓶	瓶	瓶	瓶  
0	0	0	0	// 药瓶0  
0	0	0	1	// 药瓶1  
0	0	1	0	// 药瓶2    
0	0	1	1	// 药瓶3  
0	1	0	0	// 药瓶4  
0	1	0	1	// 药瓶5  
0	1	1	0	// 药瓶6  
0	1	1	1	// 药瓶7  
1	0	0	0	// 药瓶8

输入：

整数`m`为行数，`n`为列数，`H`为梯子高度，`s`为（横向）允许跳跃次数，二维数组`map`为山洞地形。如

> 1
5 5 3 1

###--
----#
--#-#
###--

思路：

借助并查集。首先不考虑跳跃，将山洞地形按照访问规则建立起并查集，左下角元素`map[m - 1][n]`所在集合就是能访问到的。若`s = 0`，这就ok了。

1. 数串拼接成的最大数值
设有`n`个正整数，将它们连成一排，组成一个最大的多位整数
例如：
[13, 312, 343] 连接成的最大整数为 34331213
[7, 13, 4, 246] => 7424613
[98, 732, 4210, 42, 19] => 9873242421019

思路：
把数串从大到小排序。排序策略：`[0, min{len(A), len(B)}]`部分遵循字典序；当两个字符串有一个比到头了，就把没比到头的多出来的部分，重新跟已经比到头的那个比较。

比如，`785`和`78`比，比完`"78"`这部分，再拿`5`重新跟`7`比；

又如，`7878789`和`78`比，就是比了三遍`78`这段，再拿`9`跟`7`比。

 寻找一个字符串中最长的重复子串。

关于重复子串的理解：设原字符串 `str` 长度为 `n`, 它的所有子串为集合 `S = {s0, s1, s2, ..., s(n*(n+1)/2)}`, 如果存在子串 `si`与`sj`相等（`si.equals(sj) == true`）且 `i ≠ j`, 那么 `si` 与 `sj` 就是一组重复子串。

关于子串重叠的解：这里认为是允许子串有重叠的（比如原串为 `asdfasdfas`, `si` 为前缀 `asdfas`, `sj` 为后缀 `asdfas`, 它们重叠了两个字符），因为这里的 `si` 和 `sj` 完全符合上述对子串的定义。

《编程珠玑》上的解法：

>
1. 用字符串指针数组保存用户输入的字符串的所有后缀字符串；
2. 将后缀字符串集合进行排序；
3. 比较相邻字符串的公共子串长度，找到长度最大值，保存相应字符串即为所求

为什么可以这么做——

先考虑暴力解法：枚举 n*(n+1)/2 个子串，求其中的重复串。

事实上可以不用枚举 n*(n+1)/2 个那么多，n 个就够了。

`prefix(suffix(input))` 得到的就是 `input` 的所有子串

题解就是把子串与子串之间的比较转化到了后缀（也是一个子串）与后缀（它的那些前缀子串的代表）的比较
